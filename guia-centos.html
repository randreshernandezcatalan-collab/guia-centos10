// Sistema Base
Actualización del sistema
Lo primero siempre es actualizar el sistema para aplicar todos los parches de seguridad disponibles antes de instalar cualquier servicio.
CMD Actualizar paquetes del sistema
PASO 01
dnf es el gestor de paquetes de CentOS 10 (sucesor de yum). Con update descarga e instala las versiones más recientes de todo lo instalado. La flag -y responde "sí" automáticamente a todas las confirmaciones.
bash 
sudo dnf update -y # Actualiza la lista de paquetes e instala updates
sudo dnf upgrade -y # Aplica actualizaciones mayores si las hay
💡 Después de una actualización mayor, es recomendable reiniciar el sistema con sudo reboot para aplicar cambios en el kernel.
02
// Sistema Base
Nombre del servidor (Hostname)
El hostname identifica al servidor en la red. Aparece en el prompt de la terminal y en registros de log.
CMD Configurar hostname permanente
PASO 02
hostnamectl es la herramienta de systemd para gestionar el nombre del host. El cambio es permanente y no requiere reinicio.
bash
# Establecer el nombre del servidor
sudo hostnamectl set-hostname mi-servidor

# Verificar el cambio
hostnamectl status
📝 El nuevo hostname aparecerá en el prompt: [root@mi-servidor ~]$. Si no cambia de inmediato, cierra y abre una nueva sesión de terminal.

Zona horaria
La hora correcta es crítica para los logs del sistema, certificados SSL, autenticación y sincronización de servicios.
CMD Configurar zona horaria Chile
PASO 03
timedatectl gestiona la hora del sistema. La zona America/Santiago corresponde a Chile Continental (UTC-4 en verano, UTC-3 en invierno con horario de verano).
bash
# Ver todas las zonas disponibles para América
timedatectl list-timezones | grep America

# Establecer zona horaria de Chile
sudo timedatectl set-timezone America/Santiago

# Verificar configuración actual
timedatectl

// Sistema Base
Usuario administrador
Nunca se debe trabajar siempre como root. Se crea un usuario normal con permisos de sudo para administrar el servidor de forma más segura.
CMD Crear usuario con privilegios sudo
PASO 04
En CentOS el grupo wheel equivale a sudoers — cualquier usuario en ese grupo puede ejecutar comandos como root usando sudo.
bash
# Crear usuario con directorio home y shell bash
sudo useradd -m -s /bin/bash tuusuario

# Asignar contraseña
sudo passwd tuusuario

# Agregar al grupo wheel (sudo)
sudo usermod -aG wheel tuusuario

# Verificar que está en el grupo
groups tuusuario
⚠️ Reemplaza tuusuario con el nombre real que quieres usar. Usa una contraseña fuerte: mínimo 12 caracteres con mayúsculas, números y símbolos.

// Sistema Base
SSH seguro
SSH es el protocolo para acceder remotamente al servidor. Configurarlo correctamente es una de las medidas de seguridad más importantes.
CONFIG Configurar /etc/ssh/sshd_config
PASO 05
El archivo sshd_config controla el comportamiento del servidor SSH. Desactivar el login directo como root es lo mínimo indispensable.
bash
sudo nano /etc/ssh/sshd_config
📄 /etc/ssh/sshd_config — parámetros a modificar 
PermitRootLogin         no       # Deshabilitar login directo como root
PasswordAuthentication  yes      # Mantener por ahora, luego usar llaves
Port                    22       # Puerto estándar SSH
MaxAuthTries            3        # Máximo 3 intentos de contraseña
LoginGraceTime          30       # 30 segundos para autenticarse
bash
# Reiniciar SSH para aplicar cambios
sudo systemctl restart sshd

# Verificar que está activo
sudo systemctl status sshd

// Red y Seguridad
Firewall — firewalld
CentOS usa firewalld como gestor de firewall. Controla qué tráfico entra y sale del servidor por puerto y protocolo.
CMD Activar y configurar firewalld
PASO 06
firewalld trabaja con "zonas" y "servicios". La zona public es la predeterminada. Los servicios http y https abren los puertos 80 y 443 respectivamente. La flag --permanent hace los cambios persistentes entre reinicios.
bash
# Activar y habilitar al inicio
sudo systemctl enable --now firewalld

# Ver estado actual y reglas
sudo firewall-cmd --list-all

# Abrir puertos para web
sudo firewall-cmd --permanent --add-service=ssh
sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --permanent --add-service=https

# Aplicar cambios permanentes
sudo firewall-cmd --reload

# Verificar servicios habilitados
sudo firewall-cmd --list-services
💡 Durante el laboratorio se puede usar iptables -F para limpiar reglas temporalmente si algo bloquea el tráfico. En producción esto no es recomendable.
ServicioPuertoProtocoloPara qué
ssh22TCPAcceso remoto al servidor
http80TCPWeb sin cifrar
https443TCPWeb con SSL/TLS

// Red y Seguridad
SELinux
SELinux es un módulo de seguridad del kernel Linux que agrega control de acceso obligatorio (MAC). Viene activado por defecto en CentOS.
INFO Entender y configurar SELinux
PASO 07
SELinux tiene tres modos: Enforcing (bloquea y registra), Permissive (solo registra sin bloquear) y Disabled (desactivado). Para aprender, Permissive es ideal porque no bloquea pero sí avisa.
bash
# Ver estado actual
sestatus
getenforce

# Cambiar a modo permissive temporalmente
sudo setenforce 0

# Configurar contexto SELinux para el directorio web
sudo semanage fcontext -a -t httpd_sys_content_t "/var/www/misitio(/.*)?"
sudo restorecon -Rv /var/www/misitio

# Verificar contextos aplicados
ls -Z /var/www/misitio
📄 /etc/selinux/config — modo permanente 
SELINUX=permissive   # enforcing | permissive | disabled
SELINUXTYPE=targeted

Instalación de Apache (httpd)
Apache HTTP Server es el servidor web más usado del mundo. En CentOS el paquete se llama httpd.
CMD Instalar y activar Apache
PASO 11
enable --now hace dos cosas: habilita el servicio para que inicie con el sistema, y lo inicia inmediatamente sin necesidad de reiniciar.
bash
# Instalar Apache
sudo dnf install -y httpd

# Iniciar y habilitar en el arranque
sudo systemctl enable --now httpd

# Verificar que está corriendo
sudo systemctl status httpd

# Probar localmente
curl http://localhost
Estructura de directorios de Apache en CentOS:
/etc/httpd/ → configuración principal
  ├── conf/httpd.conf → config global
  ├── conf.d/ → configs adicionales (VirtualHosts aquí)
  └── conf.modules.d/ → módulos cargados

/var/www/html/ → raíz web por defecto
/var/log/httpd/ → logs de acceso y error
  ├── access_log
  └── error_log

// Servidor Web
Configurar VirtualHost
Un VirtualHost permite que Apache sirva múltiples sitios web desde el mismo servidor, identificándolos por nombre de dominio.
CONFIG Crear archivo de VirtualHost
PASO 13
El archivo de configuración va en /etc/httpd/conf.d/. Apache carga automáticamente todos los archivos .conf de ese directorio. La directiva Options -Indexes deshabilita el listado de directorios, lo cual es importante para la seguridad.
bash
sudo nano /etc/httpd/conf.d/misitio.conf
⚙️ /etc/httpd/conf.d/misitio.conf 
<VirtualHost *:80>
    ServerName    misitio.local
    ServerAlias   www.misitio.local
    DocumentRoot  /var/www/misitio

    <Directory /var/www/misitio>
        Options       -Indexes +FollowSymLinks
        AllowOverride All
        Require       all granted
    </Directory>

    ErrorLog   /var/log/httpd/misitio-error.log
    CustomLog  /var/log/httpd/misitio-access.log combined
</VirtualHost>
bash
# Verificar sintaxis de la configuración
sudo apachectl configtest

# Reiniciar Apache
sudo systemctl restart httpd

# Aplicar contexto SELinux correcto
sudo semanage fcontext -a -t httpd_sys_content_t "/var/www/misitio(/.*)?"
sudo restorecon -Rv /var/www/misitio

mod_ssl es el módulo de Apache que agrega soporte para HTTPS. Sin él, Apache solo puede servir tráfico HTTP sin cifrar.

# Reiniciar Apache para cargar el módulo
sudo systemctl restart httpd
16
// SSL / HTTPS
Generar certificado SSL autofirmado
Un certificado autofirmado cifra el tráfico igual que uno real, pero el navegador mostrará una advertencia porque no fue emitido por una CA reconocida. Es perfecto para laboratorio.
CMD Generar certificado y llave privada
PASO 16
El comando genera dos archivos: .crt (certificado público que se comparte) y .key (llave privada que NUNCA se comparte). RSA 2048 bits es el estándar mínimo actual.
bash
# Generar certificado autofirmado válido por 1 año
sudo openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
    -keyout /etc/httpd/ssl/misitio.key \
    -out    /etc/httpd/ssl/misitio.crt

# Asegurar permisos: solo root puede leer la llave privada
sudo chmod 600 /etc/httpd/ssl/misitio.key
sudo chmod 644 /etc/httpd/ssl/misitio.crt
📝 Al ejecutar el comando se pedirán datos del certificado. En Common Name escribe exactamente misitio.local — este campo debe coincidir con el dominio.

// SSL / HTTPS
VirtualHost con HTTPS
Se actualiza la configuración de Apache para servir el sitio por HTTPS y redirigir automáticamente el tráfico HTTP.
CONFIG Configuración completa HTTP + HTTPS
PASO 17
⚙️ /etc/httpd/conf.d/misitio.conf — configuración final 
# ── Bloque HTTP: redirige todo a HTTPS ──
<VirtualHost *:80>
    ServerName   misitio.local
    ServerAlias  www.misitio.local
    Redirect permanent / https://misitio.local/
</VirtualHost>

# ── Bloque HTTPS: sirve el sitio con SSL ──
<VirtualHost *:443>
    ServerName   misitio.local
    ServerAlias  www.misitio.local
    DocumentRoot /var/www/misitio

    # Activar SSL y apuntar a los certificados
    SSLEngine            on
    SSLCertificateFile   /etc/httpd/ssl/misitio.crt
    SSLCertificateKeyFile /etc/httpd/ssl/misitio.key

    # Solo protocolos modernos (deshabilitar TLS 1.0 y 1.1)
    SSLProtocol   all -SSLv3 -TLSv1 -TLSv1.1
    SSLCipherSuite HIGH:!aNULL:!MD5:!3DES

    <Directory /var/www/misitio>
        Options       -Indexes +FollowSymLinks
        AllowOverride All
        Require       all granted
    </Directory>

    ErrorLog  /var/log/httpd/misitio-ssl-error.log
    CustomLog /var/log/httpd/misitio-ssl-access.log combined
</VirtualHost>
bash
# SELinux: permitir Apache con SSL
sudo setsebool -P httpd_can_network_connect on
sudo semanage fcontext -a -t cert_t "/etc/httpd/ssl(/.*)?"
sudo restorecon -Rv /etc/httpd/ssl

# Verificar sintaxis y reiniciar
sudo apachectl configtest
sudo systemctl restart httpd

# Verificar que escucha en ambos puertos
sudo ss -tlnp | grep httpd

// Red Local
Configurar DNS local (/etc/hosts)
Para acceder al servidor por nombre en lugar de IP, se edita el archivo hosts en cada máquina cliente. Es la solución más simple para laboratorio.
CONFIG Agregar entradas en /etc/hosts
PASO 19
El archivo /etc/hosts es consultado antes que el DNS real. Si el nombre está ahí, el sistema lo resuelve localmente sin consultar ningún servidor DNS.
🐧 En Kali Linux — /etc/hosts 
sudo nano /etc/hosts

# Agregar al final:
192.168.X.X  misitio.local
192.168.X.X  www.misitio.local
🪟 En Windows — hosts 
# Abrir como Administrador:
C:\Windows\System32\
  drivers\etc\hosts

# Agregar al final:
192.168.X.X  misitio.local
bash
# Verificar que resuelve correctamente
ping misitio.local

# Probar el sitio por nombre
curl -k https://misitio.local

# Escanear con whatweb para ver el stack
whatweb https://misitio.local
💡 Reemplaza 192.168.X.X con la IP real de tu servidor CentOS. Puedes verla con ip a en el servidor.
