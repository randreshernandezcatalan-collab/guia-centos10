01
// Conceptos
¿Qué es X11?
X11 (también llamado X Window System) es el sistema que usa Linux para mostrar interfaces gráficas. Permite que los programas muestren ventanas, botones y menús en pantalla.
INFO El problema que resuelve
CONCEPTO 01
Cuando te conectas a un servidor Linux por SSH normalmente solo tienes acceso a texto. X11 Forwarding permite ejecutar programas gráficos en el servidor y verlos en tu pantalla local.
❌ Sin X11 Forwarding
Diseñador ──── SSH ────► CentOS → solo terminal de texto

✅ Con X11 Forwarding
Diseñador ──── SSH+X11 ────► CentOS → ejecuta GIMP, Firefox, gedit...
                          ◄── ventana gráfica se muestra localmente
Sin X11Con X11
Solo comandos de textoAplicaciones con ventanas
No se puede ver interfaz gráficaGIMP, Firefox, gedit visibles
El programa corre en el servidorEl programa corre en el servidor
—La pantalla se ve en el cliente
02
// Conceptos
¿Cómo funciona?
X11 funciona con arquitectura cliente-servidor, pero al revés de lo que parece intuitivo.
INFO Arquitectura cliente-servidor de X11
CONCEPTO 02
Servidor X11 → el que MUESTRA las ventanas → está en el PC del diseñador
Cliente X11 → el que EJECUTA los programas → está en CentOS

Flujo:
CentOS ── ejecuta app ──► envía gráficos por SSH ──► Pantalla diseñador
Diseñador ── hace clic ──► envía evento por SSH ──► CentOS procesa
📝 El procesamiento ocurre 100% en el servidor. El diseñador solo ve la pantalla y envía clics/teclado. El ancho de banda necesario es bajo porque solo se transmiten los gráficos, no los datos.
03
// Servidor CentOS
Instalar paquetes X11
Se instalan los paquetes mínimos necesarios para habilitar X11 en el servidor. No se instala un escritorio completo, solo el soporte para el forwarding.
CMD Instalar paquetes X11 en CentOS
PASO 03
Solo se necesitan tres paquetes en el servidor. No es necesario instalar un entorno gráfico completo como GNOME o KDE.
bash
sudo dnf install -y \
  xorg-x11-xauth \    # Maneja autenticación del túnel X11 por SSH
  xorg-x11-fonts-* \  # Fuentes necesarias para apps gráficas
  xorg-x11-utils \    # Utilidades básicas de diagnóstico X11
  xterm               # Terminal gráfica para verificar funcionamiento
PaqueteFunción¿Obligatorio?
xorg-x11-xauthAutenticación del canal X11Sí
xorg-x11-fonts-*Fuentes para aplicaciones gráficasRecomendado
xorg-x11-utilsHerramientas de diagnósticoOpcional
xtermTerminal gráfica de pruebaOpcional
04
// Servidor CentOS
Configurar SSH para X11
Se habilita el X11 Forwarding en la configuración del servidor SSH. Por defecto viene desactivado en CentOS 10.
CONFIG Editar /etc/ssh/sshd_config
PASO 04
bash
sudo nano /etc/ssh/sshd_config
📄 /etc/ssh/sshd_config — parámetros X11 
X11Forwarding     yes        # Habilitar X11 Forwarding
X11DisplayOffset  10         # Número de display X11 a usar
X11UseLocalhost   yes        # Bind X11 solo en localhost (más seguro)
bash
# Guardar y reiniciar SSH
sudo systemctl restart sshd

# Verificar que el servicio sigue activo
sudo systemctl status sshd
⚠️ Mantener X11UseLocalhost yes es importante por seguridad. Con no el display X11 queda expuesto en todas las interfaces de red.
05
// Servidor CentOS
Verificar configuración del servidor
Confirmar que SSH tiene X11 habilitado antes de intentar conectarse desde el cliente.
CMD Verificar parámetros SSH activos
PASO 05
bash
# Verificar que X11 está habilitado en la configuración activa
sudo sshd -T | grep -i x11

# Debe mostrar:
# x11forwarding yes
# x11displayoffset 10
# x11uselocalhost yes

# Verificar que xauth está instalado
which xauth
# Debe mostrar: /usr/bin/xauth
💡 Si xauth no aparece instalado, X11 Forwarding no funcionará aunque SSH esté configurado. Asegúrate de haber instalado el paquete xorg-x11-xauth.
06
// Cliente
Conectarse desde Linux / Kali
Linux ya incluye soporte X11 nativo. Solo se necesita agregar un flag al comando SSH.
CMD SSH con X11 desde Linux
PASO 06
El flag -X habilita X11 Forwarding con restricciones de seguridad. El flag -Y confía completamente en el cliente (más permisos, menos seguro). Para el laboratorio cualquiera funciona.
bash
# Conectarse con X11 habilitado (modo seguro)
ssh -X usuario@192.168.X.X

# O con modo extendido (más compatibilidad)
ssh -Y usuario@192.168.X.X

# Una vez conectado, verificar la variable DISPLAY
echo $DISPLAY
# Debe mostrar algo como: localhost:10.0

# Probar abriendo una app gráfica
xterm &
xclock &
FlagModoSeguridadUso recomendado
-XSeguroAltaProducción / red externa
-YConfianza totalMediaLaboratorio / red local

CMD Opción A — MobaXterm (recomendado)
PASO 07A
MobaXterm incluye SSH y servidor X11 integrados. No requiere configuración adicional — X11 Forwarding está activo por defecto.
pasos
1. Descargar MobaXterm desde: mobaxterm.mobatek.net
2. Instalar y abrir MobaXterm
3. Session → SSH
4. Remote host: 192.168.X.X
5. Username: tuusuario
6. En Advanced SSH settings:
   ✅ X11-Forwarding (ya viene marcado)
7. Click OK y conectar
CMD Opción B — VcXsrv + PuTTY
PASO 07B
pasos
# Paso 1: Instalar VcXsrv
Descargar desde: sourceforge.net/projects/vcxsrv
Instalar y ejecutar XLaunch:
  - Multiple windows
  - Start no client
  - ✅ Disable access control
  - Finish

# Paso 2: Configurar PuTTY
PuTTY → Connection → SSH → X11:
  ✅ Enable X11 forwarding
  X display location: localhost:0

# Paso 3: Conectar y verificar
echo $DISPLAY   # debe mostrar localhost:10.0

// Cliente
Conectarse desde macOS
macOS necesita XQuartz como servidor X11. Antes venía incluido pero Apple lo eliminó en OS X Mountain Lion.
CMD Instalar XQuartz y conectar
PASO 08
bash
# Instalar XQuartz con Homebrew
brew install --cask xquartz

# O descargar desde: xquartz.macosforge.org

# Reiniciar macOS después de instalar XQuartz

# Conectar al servidor con X11
ssh -X usuario@192.168.X.X

# Verificar
echo $DISPLAY
💡 En macOS es necesario reiniciar después de instalar XQuartz para que la variable de entorno DISPLAY se configure correctamente.

// Verificación
Pruebas de funcionamiento
Comandos para verificar que X11 Forwarding está funcionando correctamente desde el cliente.
CMD Verificar y probar aplicaciones gráficas
PASO 09
bash
# Una vez conectado por SSH -X, verificar DISPLAY
echo $DISPLAY
# Esperado: localhost:10.0 (o similar)

# Prueba 1: Terminal gráfica (más simple)
xterm &

# Prueba 2: Reloj gráfico
xclock &

# Prueba 3: Información del display
xdpyinfo | head -20

# Instalar y probar editor gráfico
sudo dnf install -y gedit
gedit &

# Instalar y probar GIMP
sudo dnf install -y gimp
gimp &
⚠️ Si $DISPLAY está vacío, X11 Forwarding no está activo. Verifica que usaste ssh -X o ssh -Y al conectarte.

// Seguridad y Tips
Seguridad y recomendaciones
Consideraciones importantes para usar X11 Forwarding de forma segura y eficiente.
INFO Consideraciones de seguridad
PASO 10
AspectoRecomendación
Usar -X vs -Y-X es más seguro. Usar -Y solo en red local de confianza
X11UseLocalhostMantener en yes para que el display no quede expuesto
Red de usoIdeal para red local. No usar sobre internet sin VPN
FirewallNo requiere puertos extra. Todo va por SSH (puerto 22)
RendimientoLento en redes con alta latencia. Mejor en red local
📝 Para producción con muchos usuarios conectándose remotamente, considera alternativas como xrdp (escritorio remoto RDP) o VNC que son más eficientes para uso intensivo.
